Câu 1: position: absolute và relative

Trả lời:

• position: relative: Định vị tương đối so với vị trí ban đầu của chính nó. Phần tử vẫn chiếm chỗ trong luồng văn bản (document flow).

• position: absolute: Định vị tuyệt đối, phần tử bị rút ra khỏi luồng văn bản và định vị dựa trên phần tử cha gần nhất có position: relative (hoặc absolute/fixed).

Ví dụ mã nguồn:

.parent { position: relative; width: 300px; height: 300px; }
.child { position: absolute; top: 0; right: 0; }

Câu 2: Domain, Hosting, DNS

Quy trình triển khai website:

  • Mua Domain: Địa chỉ định danh (ví dụ: google.com).
  • Thuê Hosting: Không gian lưu trữ mã nguồn trên Server.
  • Trỏ DNS: Hệ thống phân giải tên miền thành địa chỉ IP của Server.
  • Upload: Dùng FTP hoặc Git để đẩy code lên Hosting.

Rủi ro bảo mật: Thiếu SSL (HTTPS) dẫn đến dữ liệu bị nghe lén, lộ API Key trong mã nguồn JS công khai.

Câu 3: Phân biệt var, let, const

Bảng so sánh:

  • var: Có cơ chế hoisting (đưa khai báo lên đầu), phạm vi hàm (function scope).
  • let: Phạm vi khối (block scope {}), không thể khai báo lại trong cùng một khối.
  • const: Giống let nhưng dùng cho hằng số (giá trị không thể gán lại).

Lưu ý Debug: Luôn mở F12 (Console) để kiểm tra lỗi ReferenceError khi dùng let/const trước khi khai báo.

Câu 4: Event Delegation (Ủy quyền sự kiện)

Khái niệm: Thay vì gắn listener cho từng phần tử con (như hàng trăm thẻ LI), ta chỉ gắn 1 listener duy nhất vào thẻ cha (UL).

Mã xử lý mẫu:

document.querySelector(".parent").addEventListener("click", function(e) {
  if(e.target && e.target.matches("li.item")) {
    console.log("Bạn vừa click vào: " + e.target.innerText);
  }
});

Lợi ích: Giảm bộ nhớ, xử lý được cả các phần tử con được thêm vào sau này bằng JavaScript.
